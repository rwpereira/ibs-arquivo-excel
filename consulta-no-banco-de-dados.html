Consulta no banco de dados

De acordo com a solicitação, podemos começar selecionando os campos que são solicitados. Para isso precisamos identificar as tabelas relacionadas e indicar o nome dos campos.

 

select p.id, p.nome, concat(p2.nome, ' ', p2.sobrenome) as produtor,

      p.quantidade_funcionarios, e2.nome as estado,

      m.nome as municipio, p.area_total, p.criado_em, l.latitude,

      l.longitude,

      (select count(*)

         from atendimento a

        where a.atendimento_tipo_id = 1

          and a.propriedade_id = p.id

          and a.desativado_em is null) as atendimentos,

from propriedade p

      inner join pessoa p2 on p2.id = p.produtor_id

      inner join endereco e on e.id = p.endereco_id

      inner join municipio m on m.id = e.municipio_id

      inner join estado e2 on e2.id = m.estado_id

      inner join certificadora_propriedade cp on cp.propriedade_id = p.id

      left join localizacao l on l.id = p.localizacao_id

 

Observe que a última coluna é na verdade uma sub-consulta na tabela de atendimentos para buscar quantos atendimentos do tipo Diagnóstico a propriedade teve.

 

 

Posteriormente podemos adicionar as condições, tanto as de filtro que o usuário irá informar pelo sistema, quanto às demais que serão fixas independente de o usuário selecionar ou não.

where ((:estadoId =  e2.id) or (:estadoId is null))

  and ((:municipioId = m.id) or (:municipioId is null))

  and cp.certificadora_id = :certificadoraId

  and p.desativado_em is null

  and p2.desativado_em is null

 

Note que temos 3 parâmetros (municipioId, estadoId, certificadoraId), sendo que esses parâmetros serão passados do sistema para o nosso SQL,  e perceba que tanto o estadoId quanto o municipioId, podem ser iguais ao que foi passado no parâmetro, ou então que o parâmetro seja nulo, fazendo com que o se o usuário informar o parâmetro seja filtrado pelo parâmetros, ou se não informado que traga tudo.

order by coalesce(p.quantidade_funcionarios,0) desc
